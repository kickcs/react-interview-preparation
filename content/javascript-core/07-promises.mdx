<Answer lang="en">

A **Promise** is an object representing the eventual completion or failure of an asynchronous operation. A promise is always in one of three states: **pending** (initial state), **fulfilled** (operation completed successfully), or **rejected** (operation failed). Once a promise is settled (fulfilled or rejected), it cannot change state — this immutability makes promises more reliable than callbacks.

Promises are consumed using **`.then()`** for success, **`.catch()`** for errors, and **`.finally()`** for cleanup that runs regardless of the outcome. Promises can be **chained** — each `.then()` returns a new promise, allowing sequential asynchronous operations without nesting. If you return a value from `.then()`, the next `.then()` receives it; if you return a promise, the chain waits for it to settle.

The `Promise` class also provides powerful static methods: **`Promise.all()`** waits for all promises and fails fast on the first rejection, **`Promise.allSettled()`** waits for all regardless of outcome, **`Promise.race()`** resolves or rejects with the first settled promise, and **`Promise.any()`** resolves with the first fulfilled promise (ignoring rejections). These combinators are essential for managing concurrent asynchronous operations efficiently.

</Answer>
<Answer lang="ru">

**Promise (промис)** — это объект, представляющий результат асинхронной операции, которая может завершиться в будущем. Промис всегда находится в одном из трёх состояний: **pending** (ожидание, начальное состояние), **fulfilled** (выполнен успешно) или **rejected** (отклонён, произошла ошибка). После того как промис перешёл в конечное состояние (fulfilled или rejected), он не может его изменить — эта неизменяемость делает промисы надёжнее колбэков.

Промисы используются через **`.then()`** для успешного результата, **`.catch()`** для обработки ошибок и **`.finally()`** для очистки, которая выполняется независимо от результата. Промисы можно **объединять в цепочки** — каждый `.then()` возвращает новый промис, что позволяет выполнять последовательные асинхронные операции без вложенности. Если вы возвращаете значение из `.then()`, следующий `.then()` получит его; если вы возвращаете промис, цепочка дождётся его завершения.

Класс `Promise` также предоставляет мощные статические методы: **`Promise.all()`** ждёт выполнения всех промисов и немедленно отклоняется при первой ошибке, **`Promise.allSettled()`** ждёт завершения всех независимо от результата, **`Promise.race()`** разрешается или отклоняется с первым завершившимся промисом, а **`Promise.any()`** разрешается с первым успешно выполненным промисом (игнорируя отклонения). Эти комбинаторы необходимы для эффективного управления параллельными асинхронными операциями.

</Answer>

```javascript
// Creating a promise
const fetchData = (url) =>
  new Promise((resolve, reject) => {
    setTimeout(() => {
      if (url) resolve({ data: "result" });
      else reject(new Error("URL is required"));
    }, 1000);
  });

// Chaining
fetchData("/api/users")
  .then((response) => {
    console.log(response.data); // "result"
    return fetchData("/api/posts");
  })
  .then((response) => console.log(response.data))
  .catch((error) => console.error(error.message))
  .finally(() => console.log("Done"));

// Promise combinators
const p1 = Promise.resolve(1);
const p2 = Promise.resolve(2);
const p3 = Promise.reject("error");

Promise.all([p1, p2]).then(console.log);         // [1, 2]
Promise.allSettled([p1, p3]).then(console.log);
// [{ status: "fulfilled", value: 1 }, { status: "rejected", reason: "error" }]
Promise.race([p1, p2]).then(console.log);         // 1 (first settled)
Promise.any([p3, p2]).then(console.log);          // 2 (first fulfilled)
```
