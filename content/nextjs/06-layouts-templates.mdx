<AnswerGroup>
<Answer lang="en">

**Layouts** in Next.js are defined by `layout.tsx` files and wrap the content of their child routes. The **root layout** (`app/layout.tsx`) is required and must include `<html>` and `<body>` tags. Layouts are shared between pages and **preserve state** across navigations — they don't re-render or unmount when the user navigates between sibling routes. This makes them ideal for persistent UI like navigation bars, sidebars, and footers.

**Nested layouts** allow you to create layout hierarchies. Each route segment can have its own `layout.tsx`, and they compose automatically — a child layout wraps its pages and is itself wrapped by the parent layout. Layouts receive a `children` prop containing the child page or nested layout. They can also be async Server Components that fetch their own data.

**Templates** (`template.tsx`) are similar to layouts but **re-mount on every navigation**. A new instance is created each time the user navigates, which means state is reset, effects re-run, and the DOM is recreated. Templates are useful when you need entry animations, per-page logging, or per-navigation form resets. You can also export **metadata** directly from layouts and pages using the `metadata` object or `generateMetadata` function to define titles, descriptions, and other meta tags for each route segment.

</Answer>
<Answer lang="ru">

**Макеты** (Layouts) в Next.js определяются файлами `layout.tsx` и оборачивают содержимое дочерних маршрутов. **Корневой макет** (`app/layout.tsx`) обязателен и должен включать теги `<html>` и `<body>`. Макеты разделяются между страницами и **сохраняют состояние** при навигации — они не перерендериваются и не размонтируются при переходе между соседними маршрутами. Это делает их идеальными для постоянного UI: навигации, боковых панелей и футеров.

**Вложенные макеты** позволяют создавать иерархии макетов. Каждый сегмент маршрута может иметь свой `layout.tsx`, и они компонуются автоматически — дочерний макет оборачивает свои страницы и сам оборачивается родительским макетом. Макеты получают проп `children`, содержащий дочернюю страницу или вложенный макет. Они также могут быть асинхронными серверными компонентами, загружающими собственные данные.

**Шаблоны** (`template.tsx`) похожи на макеты, но **перемонтируются при каждой навигации**. Новый экземпляр создаётся каждый раз при переходе, что означает сброс состояния, повторный запуск эффектов и пересоздание DOM. Шаблоны полезны для входных анимаций, логирования по страницам или сброса форм при навигации. Также можно экспортировать **метаданные** прямо из макетов и страниц, используя объект `metadata` или функцию `generateMetadata` для определения заголовков, описаний и других мета-тегов для каждого сегмента маршрута.

</Answer>

```tsx
// app/layout.tsx — Root layout (required)
import { ReactNode } from "react";

export const metadata = {
  title: "My App",
  description: "Built with Next.js",
};

export default function RootLayout({ children }: { children: ReactNode }) {
  return (
    <html lang="en">
      <body>
        <nav>
          <a href="/">Home</a>
          <a href="/dashboard">Dashboard</a>
        </nav>
        <main>{children}</main>
        <footer>© 2024 My App</footer>
      </body>
    </html>
  );
}
```

```tsx
// app/dashboard/layout.tsx — Nested layout (persists across child routes)

export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <div style={{ display: "flex" }}>
      <aside>
        <h2>Dashboard</h2>
        <ul>
          <li><a href="/dashboard">Overview</a></li>
          <li><a href="/dashboard/analytics">Analytics</a></li>
          <li><a href="/dashboard/settings">Settings</a></li>
        </ul>
      </aside>
      <section style={{ flex: 1 }}>{children}</section>
    </div>
  );
}

// app/dashboard/page.tsx → /dashboard
// app/dashboard/analytics/page.tsx → /dashboard/analytics
// Both share the sidebar from DashboardLayout
```

```tsx
// app/dashboard/template.tsx — Re-mounts on every navigation
"use client";

import { useEffect } from "react";

export default function DashboardTemplate({
  children,
}: {
  children: React.ReactNode;
}) {
  useEffect(() => {
    // Runs on every navigation (unlike layout)
    console.log("Page viewed:", window.location.pathname);
  }, []);

  return <div className="fade-in">{children}</div>;
}
```

```txt
Rendering order (outermost to innermost):

app/layout.tsx          → Root layout (persists always)
  app/dashboard/layout.tsx    → Dashboard layout (persists within /dashboard)
    app/dashboard/template.tsx  → Template (re-mounts on navigation)
      app/dashboard/*/page.tsx  → Page content (changes per route)
```
