---
title: "Оператор +: конкатенация vs арифметика"
order: 6
---

Оператор `+` в JavaScript перегружен: он либо складывает числа, либо конкатенирует строки — в зависимости от типов операндов. Это приводит к серии знаменитых «WTF JS» результатов.

**Предскажите, что выведет каждый `console.log`:**

<StarterCode>

```js
console.log([] + []);
console.log([] + {});
console.log({} + []);
console.log("5" + 3);
console.log("5" - 3);
console.log(+"");
```

</StarterCode>

<Hint title="Подсказка">
Если хотя бы один операнд — строка, `+` выполняет конкатенацию. Массивы и объекты приводятся к примитиву через `ToPrimitive`: массив `[]` даёт пустую строку `""`, объект `{}` даёт строку `"[object Object]"`. Но `{} + []` в начале строки — это блок кода, а не объект!
</Hint>

<Solution>

Вывод:
```
""
"[object Object]"
0
"53"
2
0
```

**Объяснение:**

- `[] + []` → `""`: оба массива приводятся к строке. `[].toString()` даёт `""`. Итого `"" + ""` = `""`.

- `[] + {}` → `"[object Object]"`: `[]` → `""`, `{}` → `"[object Object]"`. Итого `"" + "[object Object]"` = `"[object Object]"`.

- `{} + []` → `0`: когда `{}` стоит **в начале выражения**, JS парсит его как **пустой блок кода**, а не объект. Поэтому выражение читается как: блок `{}`, затем унарный `+` применяется к `[]`. `+[]` → `+""` → `0`.

- `"5" + 3` → `"53"`: число `3` приводится к строке `"3"`, выполняется конкатенация.

- `"5" - 3` → `2`: оператор `-` не поддерживает строки, поэтому `"5"` приводится к числу `5`, выполняется арифметика. `5 - 3 = 2`.

- `+""` → `0`: унарный `+` конвертирует операнд в число. `Number("")` = `0`.

</Solution>
