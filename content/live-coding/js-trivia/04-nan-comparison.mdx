`NaN` — единственное значение в JavaScript, не равное самому себе. Задача проверяет знание различных способов проверки на `NaN` и их неочевидных отличий.

**Предскажите, что выведет каждый `console.log`:**

<StarterCode>

```js
console.log(NaN === NaN);
console.log(NaN !== NaN);
console.log(isNaN("hello"));
console.log(Number.isNaN("hello"));
console.log(Object.is(NaN, NaN));
console.log(Object.is(0, -0));
```

</StarterCode>

<Hint title="Подсказка">
Глобальная функция `isNaN()` сначала приводит аргумент к числу, а потом проверяет. `Number.isNaN()` — строгая проверка без приведения типов. `Object.is()` использует алгоритм SameValue, который отличается от `===` в двух случаях: `NaN` и `±0`.
</Hint>

<Solution>

Вывод:
```
false
true
true
false
true
false
```

**Объяснение:**

- `NaN === NaN` → `false`: по стандарту IEEE 754 NaN не равен ничему, включая самого себя. Это единственное значение в JS, нарушающее рефлексивность равенства.

- `NaN !== NaN` → `true`: следствие предыдущего.

- `isNaN("hello")` → `true`: глобальная `isNaN()` сначала вызывает `Number("hello")`, получает `NaN`, затем проверяет `NaN` — возвращает `true`. Это ловушка: `isNaN("hello")` не говорит нам, что `"hello"` это NaN, просто что после приведения получается NaN.

- `Number.isNaN("hello")` → `false`: строгая проверка без приведения типов. `"hello"` — строка, не NaN, поэтому `false`. `Number.isNaN()` вернёт `true` только если аргумент буквально является значением `NaN`.

- `Object.is(NaN, NaN)` → `true`: `Object.is` использует алгоритм SameValue (не `===`). Он специально определяет `NaN` равным самому себе.

- `Object.is(0, -0)` → `false`: другое отличие от `===`. В JS существует `+0` и `-0`. Оператор `===` считает их равными (`0 === -0` → `true`), но `Object.is` различает их.

</Solution>
