---
title: "setTimeout в цикле: var vs let"
order: 8
---

Классическая задача на замыкания и область видимости. Покажите разницу между `var` и `let` при использовании асинхронного кода внутри цикла.

**Предскажите, что выведет каждый `console.log`:**

<StarterCode>

```js
// Цикл с var
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log('var:', i), 0);
}

// Цикл с let
for (let j = 0; j < 3; j++) {
  setTimeout(() => console.log('let:', j), 0);
}
```

</StarterCode>

<Hint title="Подсказка">
`var` создаёт одну переменную на всю функцию (или глобальную область). К моменту срабатывания `setTimeout`, цикл уже завершён и `i === 3`. `let` создаёт новую переменную `j` для каждой итерации цикла — каждый колбэк захватывает своё значение.
</Hint>

<Solution>

Вывод:
```
var: 3
var: 3
var: 3
let: 0
let: 1
let: 2
```

**Объяснение:**

**Цикл с `var`:**
`var i` объявляется в функциональной (или глобальной) области видимости. Все три колбэка `setTimeout` замыкаются на **одну и ту же** переменную `i`. Колбэки выполняются после того, как синхронный код (включая цикл) завершится. К этому моменту `i` уже равно `3` (условие `i < 3` стало ложным при `i === 3`). Поэтому все три лога выводят `3`.

**Цикл с `let`:**
`let j` создаёт **новую переменную для каждой итерации** цикла. Это поведение специально определено в спецификации для `let`/`const` в `for`. Каждый колбэк замыкается на свою копию `j` со значением `0`, `1` и `2` соответственно.

**Традиционные решения до `let`:**
```js
// 1. IIFE (Immediately Invoked Function Expression)
for (var i = 0; i < 3; i++) {
  (function(i) {
    setTimeout(() => console.log(i), 0);
  })(i);
}

// 2. bind
for (var i = 0; i < 3; i++) {
  setTimeout(console.log.bind(null, i), 0);
}
```

</Solution>
