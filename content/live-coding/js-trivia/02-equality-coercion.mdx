title: "== и приведение типов"

---

Оператор нестрогого равенства `==` выполняет неявное приведение типов по запутанным правилам спецификации. Задача проверяет понимание этих правил.

**Предскажите, что выведет каждый `console.log`:**

<StarterCode>

```js
console.log('' == 0);
console.log(false == '');
console.log(null == undefined);
console.log(null == 0);
console.log(NaN == NaN);
console.log([] == false);
```

</StarterCode>

<Hint title="Подсказка">
Алгоритм `==` из спецификации: если типы совпадают — обычное сравнение; если один операнд `boolean` — он преобразуется в число; если строка и число — строка преобразуется в число; `null == undefined` — специальный случай; `null == 0` — тоже специальный случай.
</Hint>

<Solution>

Вывод:
```
true
true
true
false
false
true
```

**Объяснение:**

- `'' == 0` → `true`: строка `''` приводится к числу `0`, затем `0 == 0` → `true`.

- `false == ''` → `true`: `false` приводится к числу `0`, затем `0 == ''`; строка `''` приводится к числу `0`, затем `0 == 0` → `true`.

- `null == undefined` → `true`: это специальное правило спецификации — `null` и `undefined` равны только друг другу через `==`, и ничему больше.

- `null == 0` → `false`: несмотря на то что `Number(null) === 0`, алгоритм `==` содержит явное правило: `null` равен только `undefined` (и самому себе). С любым другим значением `null == X` вернёт `false`.

- `NaN == NaN` → `false`: NaN не равен ничему, включая самого себя. Это правило IEEE 754.

- `[] == false` → `true`: `false` приводится к `0`; `[]` приводится к примитиву через `ToPrimitive` → `''`; `'' == 0` → `''` приводится к `0`; `0 == 0` → `true`.

</Solution>
