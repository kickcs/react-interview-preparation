---
title: "Array.sort() без компаратора"
order: 5
---

Метод `Array.prototype.sort()` без аргументов сортирует элементы **лексикографически** (как строки), а не числово. Это приводит к неожиданным результатам при сортировке чисел.

**Предскажите, что выведет каждый `console.log`:**

<StarterCode>

```js
console.log([10, 9, 8, 1, 2, 3].sort());
console.log([1, 2, 11, 20, 9].sort());
console.log([3, 1, -1, 0, -2].sort());
console.log(['banana', 'apple', 'cherry'].sort());
```

</StarterCode>

<Hint title="Подсказка">
Без функции-компаратора `.sort()` приводит каждый элемент к строке и сравнивает их посимвольно. Строка `"10"` начинается с `"1"`, которая лексикографически меньше `"2"`, поэтому `10` окажется перед `2`.
</Hint>

<Solution>

Вывод:
```
[1, 10, 2, 3, 8, 9]
[1, 11, 2, 20, 9]
[-1, -2, 0, 1, 3]
["apple", "banana", "cherry"]
```

**Объяснение:**

- `[10, 9, 8, 1, 2, 3].sort()` → `[1, 10, 2, 3, 8, 9]`: числа приводятся к строкам и сравниваются посимвольно. `"1"` < `"10"` < `"2"` < `"3"` < `"8"` < `"9"`, потому что сравнение идёт слева направо по символам.

- `[1, 2, 11, 20, 9].sort()` → `[1, 11, 2, 20, 9]`: аналогично. `"11"` начинается с `"1"`, поэтому идёт перед `"2"`.

- `[3, 1, -1, 0, -2].sort()` → `[-1, -2, 0, 1, 3]`: отрицательные числа начинаются с `"-"`, код символа которого (45) меньше кода `"0"` (48). Строки `"-1"` и `"-2"` сравниваются посимвольно: вторые символы `"1"` < `"2"`, поэтому `"-1"` перед `"-2"`. Получается, что `-1` и `-2` стоят в «неправильном» порядке для математической сортировки.

- `['banana', 'apple', 'cherry'].sort()` → `["apple", "banana", "cherry"]`: строки сортируются ожидаемо в алфавитном порядке.

Для числовой сортировки всегда используйте компаратор: `.sort((a, b) => a - b)`.

</Solution>
