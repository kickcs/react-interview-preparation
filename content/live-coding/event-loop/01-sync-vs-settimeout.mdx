---
title: "Синхронный код vs setTimeout"
order: 1
---

Базовая задача на понимание разницы между синхронным кодом в call stack и колбэком в macrotask queue. Колбэк `setTimeout` никогда не выполняется раньше завершения текущего синхронного блока.

**Предскажите порядок вывода в консоль:**

<StarterCode>

```js
console.log('1');
setTimeout(() => console.log('2'), 0);
console.log('3');
```

</StarterCode>

<Hint title="Подсказка">
`setTimeout(..., 0)` не означает «выполни немедленно». Он лишь говорит: «добавь колбэк в macrotask queue после минимальной задержки». Но event loop заберёт этот колбэк только тогда, когда call stack полностью опустеет.
</Hint>

<Solution>

Порядок вывода:
```
1
3
2
```

**Объяснение:**

- **Call stack (синхронный код):** `console.log('1')` выполняется немедленно → выводит `1`. `setTimeout` регистрирует колбэк в Web API и сразу возвращает управление — `2` ещё не выводится. `console.log('3')` выполняется немедленно → выводит `3`. Call stack пуст.

- **Macrotask queue:** Event loop видит, что call stack пуст, и берёт следующую macrotask — колбэк из `setTimeout`. Он выполняется → выводит `2`.

Ключевое правило: колбэк `setTimeout` попадает в **macrotask queue** (также называемую task queue). Event loop берёт из неё задачу только после того, как call stack полностью опустеет и будет обработана вся microtask queue.

</Solution>
