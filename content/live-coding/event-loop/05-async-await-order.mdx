---
title: "async/await: порядок выполнения"
order: 5
---

`async/await` — это синтаксический сахар над промисами. Ключевое слово `await` приостанавливает выполнение async-функции и возобновляет его как микротаску после того, как awaited-выражение разрешится. Код после `await` — это фактически `.then()`-колбэк.

**Предскажите порядок вывода в консоль:**

<StarterCode>

```js
async function foo() {
  console.log('1');
  await bar();
  console.log('2');
}

function bar() {
  console.log('3');
}

console.log('4');
foo();
console.log('5');
```

</StarterCode>

<Hint title="Подсказка">
`await bar()` сначала вызывает `bar()` синхронно (поэтому `3` выводится сразу). Но `console.log('2')`, стоящий после `await`, откладывается как микротаска — он выполнится только после того, как call stack опустеет. Вызов `foo()` — синхронный, он не ждёт завершения async-функции.
</Hint>

<Solution>

Порядок вывода:
```
4
1
3
5
2
```

**Объяснение:**

1. `console.log('4')` → `4` (синхронно, до вызова `foo`)

2. `foo()` вызывается синхронно:
   - `console.log('1')` → `1`
   - `await bar()` — сначала **синхронно** вызывает `bar()`: `console.log('3')` → `3`
   - `bar()` возвращает `undefined` → `await undefined` эквивалентно `await Promise.resolve(undefined)`
   - Всё, что идёт после `await`, ставится в microtask queue. `foo()` приостанавливается и возвращает pending Promise.

3. Управление возвращается в основной call stack: `console.log('5')` → `5`. Call stack пуст.

4. **Microtask queue:** Возобновляется выполнение `foo()` после `await` → `console.log('2')` → `2`.

Важно: `await expr` всегда создаёт минимум одну микротаску — даже если `expr` уже resolved. Продолжение после `await` никогда не выполняется синхронно.

</Solution>
