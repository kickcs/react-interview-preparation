---
title: "Цепочка промисов и микротаски"

---

Каждый вызов `.then()` не выполняется мгновенно — он создаёт **новую микротаску** и ставит её в конец microtask queue. Это означает, что промисы в цепочке выполняются не все сразу, а по одному за «тик» microtask queue.

**Предскажите порядок вывода в консоль:**

<StarterCode>

```js
Promise.resolve()
  .then(() => {
    console.log('A');
    return 'B';
  })
  .then(val => console.log(val));

console.log('C');
```

</StarterCode>

<Hint title="Подсказка">
`Promise.resolve().then(cb)` не вызывает `cb` синхронно — он ставит `cb` в microtask queue. Когда первый `.then` завершается и возвращает `'B'`, второй `.then` создаёт ещё одну микротаску. `console.log('C')` — синхронный код, он выполнится до любых микротасок.
</Hint>

<Solution>

Порядок вывода:
```
C
A
B
```

**Объяснение:**

1. **Call stack:** `Promise.resolve().then(...)` — регистрирует первый колбэк в microtask queue. Сам `.then` возвращает новый промис, к которому прицеплен второй `.then` — но его колбэк пока не в очереди (второй промис ещё pending). `console.log('C')` → выводит `C`. Call stack пуст.

2. **Microtask queue, итерация 1:** Выполняется первый колбэк → выводит `A`, возвращает `'B'`. Первый промис теперь resolved со значением `'B'`. Это немедленно ставит колбэк второго `.then` в microtask queue.

3. **Microtask queue, итерация 2:** Выполняется второй колбэк → `val` равен `'B'` → выводит `B`.

Важно: microtask queue опустошается полностью (включая новые микротаски, добавленные во время обработки) перед переходом к следующей macrotask. Именно поэтому цепочка промисов не прерывается другими задачами.

</Solution>
