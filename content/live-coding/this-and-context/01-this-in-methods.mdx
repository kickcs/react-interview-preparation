---
title: "Потеря контекста this"
order: 1
---

Когда метод объекта вызывается «отдельно» — без объекта перед точкой — `this` теряется. Задача показывает три сценария: нормальный вызов метода, вызов через переменную и вызов через `setTimeout`.

**Предскажите, что выведет каждый `console.log`:**

<StarterCode>

```js
const obj = {
  name: 'Alice',
  greet() {
    console.log(this.name);
  }
};

// Вызов 1
obj.greet();

// Вызов 2
const fn = obj.greet;
fn();

// Вызов 3
setTimeout(obj.greet, 0);
```

</StarterCode>

<Hint title="Подсказка 1">
Правило неявного связывания (implicit binding): если перед вызовом функции стоит точка и объект — `this` равен этому объекту. `obj.greet()` — объект перед точкой есть, значит `this === obj`.
</Hint>

<Hint title="Подсказка 2">
Когда вы присваиваете метод переменной `const fn = obj.greet`, вы копируете ссылку на функцию без привязки к объекту. Вызов `fn()` — это вызов без объекта перед точкой. Применяется правило связывания по умолчанию (default binding): в строгом режиме `this === undefined`, в нестрогом — `this === globalThis`.
</Hint>

<Hint title="Подсказка 3">
`setTimeout(obj.greet, 0)` передаёт функцию как колбэк — точно так же, как `const fn = obj.greet; fn()`. К моменту вызова объект уже «забыт». В браузере `this` внутри колбэка `setTimeout` будет `window` (нестрогий режим), а `window.name` — пустая строка или значение по умолчанию.
</Hint>

<Solution>

Вывод (в браузере, нестрогий режим):
```
"Alice"
""        // window.name (пустая строка по умолчанию в браузере)
""        // то же самое
```

Вывод (в Node.js, нестрогий режим):
```
"Alice"
undefined // globalThis.name не определён
undefined
```

**Объяснение:**

В JavaScript `this` определяется **в момент вызова функции**, а не в момент её объявления. Существуют четыре правила привязки `this` (от низшего приоритета к высшему):

1. **Default binding (по умолчанию)** — обычный вызов `fn()`. В нестрогом режиме `this === globalThis` (`window` в браузере), в строгом (`'use strict'`) — `this === undefined`.

2. **Implicit binding (неявное)** — вызов через точку `obj.fn()`. `this === obj`.

3. **Explicit binding (явное)** — вызов через `call`/`apply`/`bind`. `this` задаётся явно.

4. **New binding** — вызов через `new`. `this === новый объект`.

**Разбор вызовов:**

- `obj.greet()` — объект перед точкой есть → **implicit binding** → `this === obj` → выводит `"Alice"`.

- `const fn = obj.greet; fn()` — ссылка скопирована, при вызове нет объекта перед точкой → **default binding** → `this === globalThis` → `this.name` это `globalThis.name`, что не равно `"Alice"`.

- `setTimeout(obj.greet, 0)` — функция передаётся как аргумент, контекст теряется так же, как в предыдущем случае.

**Как исправить:**

```js
// 1. bind — создаёт новую функцию с зафиксированным this
const fn = obj.greet.bind(obj);
fn(); // "Alice"

// 2. Стрелочная обёртка
setTimeout(() => obj.greet(), 0); // "Alice"

// 3. bind прямо в аргументе
setTimeout(obj.greet.bind(obj), 0); // "Alice"
```

</Solution>
