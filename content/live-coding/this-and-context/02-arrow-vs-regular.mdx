---
title: "Стрелочная vs обычная функция: this"
order: 2
---

Стрелочные функции не имеют собственного `this` — они захватывают его из **лексического окружения** (того места, где функция была написана). Обычные функции получают `this` при вызове.

**Предскажите, что вернут `obj.regular()` и `obj.arrow()`:**

<StarterCode>

```js
const obj = {
  value: 42,
  regular() {
    return this.value;
  },
  arrow: () => this.value
};

console.log(obj.regular()); // ?
console.log(obj.arrow());   // ?
```

</StarterCode>

<Hint title="Подсказка 1">
`regular()` — это обычный метод объекта. При вызове `obj.regular()` применяется правило неявного связывания: `this === obj`. Поэтому `this.value === obj.value`.
</Hint>

<Hint title="Подсказка 2">
`arrow: () => this.value` — стрелочная функция, написанная **в теле объектного литерала**. Объектный литерал `{ ... }` НЕ создаёт новую область видимости. `this` внутри стрелки — это `this` того места, где написан объектный литерал, то есть **внешний this**.
</Hint>

<Hint title="Подсказка 3">
Если этот код выполняется в глобальной области видимости (не внутри класса или другой функции), то `this` снаружи — это `globalThis`. `globalThis.value` не определён, поэтому стрелка вернёт `undefined`. В строгом режиме модулей `this` на верхнем уровне — `undefined`, и будет `TypeError`.
</Hint>

<Solution>

Вывод (браузер/Node.js, нестрогий режим):
```
42
undefined
```

В строгом режиме (ES-модули):
```
42
TypeError: Cannot read properties of undefined (reading 'value')
```

**Объяснение:**

**Обычная функция `regular()`:**
При вызове `obj.regular()` движок смотрит: есть ли объект перед точкой? Да, `obj`. Значит **implicit binding** → `this === obj` → `this.value === 42`.

**Стрелочная функция `arrow`:**
Стрелочные функции — особый случай. У них нет собственного `this`. Когда движок ищет `this` внутри стрелки, он идёт вверх по **лексическому окружению** (цепочке областей видимости) до первой области, где `this` определён.

Объектный литерал `{ value: 42, ... }` — это просто синтаксис. Он НЕ является областью видимости (как функция или блок `{}`). Поэтому стрелка смотрит дальше — в контекст, в котором написан весь этот код. Если это глобальный уровень, то `this === globalThis`.

**Ключевое отличие:**

| | Обычная функция | Стрелочная функция |
|---|---|---|
| `this` определяется | при **вызове** | при **написании** (лексически) |
| Можно изменить через `call`/`bind` | Да | Нет — `this` зафиксирован |
| Подходит как метод объекта | Да | Нет, если нужен `this` объекта |
| Подходит как колбэк в методе | Нет (теряет `this`) | Да (захватывает внешний `this`) |

**Типичное применение стрелки в методе:**

```js
const obj = {
  value: 42,
  getDoubled() {
    // здесь this === obj (обычный метод)
    const multiply = (n) => this.value * n; // стрелка захватывает this === obj
    return multiply(2);
  }
};

console.log(obj.getDoubled()); // 84
```

</Solution>
